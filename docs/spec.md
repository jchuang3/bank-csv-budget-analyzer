# 功能規格

## 核心功能（v0.2 — 銀行 CSV 匯入分析）

1. **CSV 上傳**：拖放或選檔，支援一次多檔案，自動辨識銀行格式（目前：AMEX、Chase）
2. **格式解析**：每種銀行格式各自的欄位對應寫在 `js/csv-parser.js` 的 `BANK_FORMATS`，要加新銀行只要新增一筆定義
3. **統一交易格式**：不論哪家銀行，都轉成 `{id, date, description, amount, bank, category}`；`amount` 一律「支出為負、收入為正」
4. **去重合併**：用 `bank+date+description+amount` 算出的 id 判斷重複，同一份 CSV 重複上傳不會產生兩筆
5. **自動分類**：規則引擎（見下方）+ 使用者手動修改後的「學習記憶」，存在 localStorage，之後同樣商家會自動套用
6. **全部交易檢視**：可搜尋、依分類/銀行/收支方向篩選
7. **月報表**：切換月份，看該月收入/支出/淨額，及支出分類長條圖（含佔當月支出百分比）
8. **大額交易**：月報表內可設定門檻（預設 $100），列出當月所有絕對值超過門檻的交易，收入支出都算
9. **交通分析**：把「交通」大分類再細分成 Uber/Lyft、Turo、租車、大眾運輸、其他交通，可看明細、可看每月各細項花費

## 資料結構

```js
// 交易
{
  id: string,
  date: "YYYY-MM-DD",
  description: string,
  amount: number,      // 正=收入，負=支出
  bank: "AMEX" | "Chase",
  category: string,
  rawCategory: string,  // 銀行原生分類（若有，如AMEX）
  type: string           // 銀行原生交易類型（若有，如Chase的Type欄位）
}
```

## 分類邏輯（js/categorizer.js）

優先順序：
1. 使用者手動改過的分類（存在 localStorage `category-overrides-v1`，用「去除數字/ID後的商家關鍵字」比對）
2. AMEX 原生 Category 欄位對應
3. 關鍵字規則（比對交易說明 + 銀行原生類型欄位）
4. 都沒中：預設「其他」

目前分類清單：薪資、轉帳、信用卡還款、房貸、社區管理費、水電瓦斯、稅務、餐飲、交通、購物、訂閱服務、醫療、退款、其他

## 設計方向

- 主題：東方筆墨感，取「記帳如記事」的意象
- 主色：墨藍 #1B2A4A、玉綠 #4A7C6E、宣紙白 #F7F4EE、朱紅 #A83C3C（用於支出/警示）
- 字體：襯線體標題 + 無襯線體內文數字（帳目數字要好讀）

## 未來可能擴充

- 支援更多銀行格式（只需擴充 `BANK_FORMATS`）
- 月與月之間的趨勢比較（折線圖）
- 匯出篩選後的結果成 CSV
- 手動新增單筆紀錄（v0.1 曾有此功能，v0.2 先移除聚焦在 CSV 匯入分析）
